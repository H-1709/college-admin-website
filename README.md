# College Admin Management System

A lightweight, modular web application designed to streamline college operations for students, faculty, staff, and administrators. 

## 🚀 Project Overview
This platform serves as a central hub for academic and administrative tasks. It is designed with a clean separation of concerns, ensuring each user role has a dedicated interface tailored to their specific needs.

## 👥 User Roles & Features

### 1. Student
* **Academic Hub:** View timetables, attendance, and internal marks.
* **Campus Life:** Stay updated with college notices and events.
* **Finance:** Track fee status and payment history.

### 2. Faculty
* **Class Management:** Manage student attendance and upload study materials.
* **Assessments:** Post marks and evaluate student performance.
* **Communication:** Broadcast announcements to specific classes.

### 3. Staff (Admission & Operations)
* **Admission Cell:** Manage new inquiries and verify applicant documents.
* **Hiring:** Post and manage job openings for faculty and staff.
* **Operations:** Handle fee records and certificate issuances.

### 4. Super Admin
* **User Control:** Manage accounts for students, faculty, and staff.
* **Institutional Setup:** Define departments, courses, and subjects.
* **Analytics:** High-level reports on enrollment and college stats.

## 📂 Folder Structure
The project follows a modular directory structure for easy scalability:

* `/public`: Entry-level pages (Login, Role Selection).
* `/admin`, `/faculty`, `/student`, `/staff`: Role-specific dashboards and modules.
* `/layouts`: Reusable HTML components (Headers, Sidebars).
* `/assets`: Styling (CSS), Logic (JS), and Media (Images).
* `/utils`: Global constants and helper functions.

## 🛠️ Tech Stack
* **Frontend:** HTML5, CSS3 (Custom Grid/Flexbox Layouts)
* **Scripting:** Vanilla JavaScript
* **Version Control:** Git & GitHub
